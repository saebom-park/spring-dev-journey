# 09. CS 정리 노트 (cs-notes)

> 이 폴더는 자바 백엔드 개발자로서 꼭 알아야 할  
> **컴퓨터 공학 핵심 개념(CS)**을 정리하는 공간이다.  
> 운영체제, 네트워크, 데이터베이스, 디자인 패턴 등 실무와 면접 모두에 필요한 개념을 **Day별 학습**으로 구성한다.  

---

## 📌 정리 목적

- **실무 + 면접 대비**를 위한 핵심 CS 지식을 체계적으로 학습  
- 단순 암기가 아니라, **이해 → 정리 → 말하기 연습**의 3단계 방식으로 내재화  
- 학습한 내용을 Day별 문서와 Practice 문서로 구분하여 관리  

---

## 🧠 주요 정리 영역

| 카테고리     | 예시 키워드 |
|--------------|--------------|
| 운영체제 (os) | 프로세스/스레드, 컨텍스트 스위칭, 메모리 관리, 스케줄링 |
| 네트워크 (network) | TCP/UDP, 3-way-handshake, HTTP/1~3, 쿠키/세션, CORS |
| 데이터베이스 (db) | 인덱스, 조인, 정규화, 트랜잭션, 락, 격리 수준 |
| 개발자 필수 지식 (dev-basic) | Git 전략, HTTP 상태코드, REST, 인코딩 |
| 디자인 패턴 (design-pattern) | 싱글톤, 팩토리, 전략, 옵저버, MVC |
| 면접 대비 (interview) | 자주 나오는 면접 질문 정리, 답변 시뮬레이션 |

## ✨ 정리 방식 (최종 기준)

- **Day 문서**: 보고서 스타일 정리  
  (정의 → 하위 구분 → 원리 → 구조 → 예상 질문 → 핵심 요약)  
- **Practice 문서**: 암기·연습 전용  
  (한 문장 요약 → 5개 포인트 → Q&A → 핵심 요약)  
- 모든 문서는 `dayNN-주제명.md`, `practiceNN-주제명.md` 네이밍 유지  
- 복습은 별도 Day로 포함하지 않고, 자율적으로 질문/답변 시뮬레이션을 통해 진행
