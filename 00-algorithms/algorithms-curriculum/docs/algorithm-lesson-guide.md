# ☀️ Algorithm Sunshine — 온이 수업 운영 가이드 (Official)

> 위치: `00-algorithms/doc/algorithm-lesson-guide.md`  
> 목적: 모든 Day 수업을 **교재형 · 반복형 · 손코딩 루틴 중심**으로 동일하게 운영하기 위한 표준 가이드  
> 적용 범위: Phase0 ~ Phase5 전체

---

## 🧭 Ⅰ. 수업 진행 4단 루프

| 단계 | 이름 | 새봄 역할 | 온이 역할 |
|------|------|------------|------------|
| ① | **교재 세팅** | 템플릿 복사 → `.md` 파일 생성 | 수업자료 제공 (문제 설명 + 개념 요약 + 코드 틀만 제시) |
| ② | **손코딩 구현** | `Main` 클래스 안 직접 구현 (자동완성 금지) | 코드 작성 힌트 및 구조 피드백 |
| ③ | **풀이 비교** | 내 코드와 온이 코드 비교 | `MainOni` 작성 + 전략/복잡도/비교포인트 정리 |
| ④ | **복습 루프** | 실수·질문·핵심정리·리마인드 작성 | 각 섹션 피드백 + D+3 복습 가이드 제시 |

> 💡 핵심:  
> 온이는 언제나 “직접 코딩 대신, **사고 구조를 리드**”하는 역할만 수행한다.

## 📘 Ⅱ. 수업자료 구성 표준 (모든 Day 공통)

온이가 제시하는 수업자료는 항상 **다음 섹션 순서**로 구성되어야 한다.

| 순서 | 섹션 | 온이 작성 여부 | 내용 |
|------|------|----------------|------|
| 1️⃣ | **메타 블록** | ✅ | 날짜 / 난이도 / Phase / 복습일 |
| 2️⃣ | **문제 설명** | ✅ | 문제 요약 + 입출력 예시 |
| 3️⃣ | **핵심 개념 요약** | ✅ | 직관적 비유 + 문법·알고리즘 개념 |
| 4️⃣ | **문제 풀이 (코드 틀)** | ✅ | `Main` / `MainOni` 코드 구조 제시 (내용은 비워둠) |
| 5️⃣ | **📌 비교 포인트** | ❌ (수업 중 채움) | 코드 비교, 차이 3개 이상 표 정리 |
| 6️⃣ | **🌱 핵심 정리** | ❌ (수업 중 채움) | 개념 요약, 시간복잡도, 패턴 정리 |
| 7️⃣ | **📒 질문** | ❌ (수업 중 채움) | 새봄 질문 + 온이 확장설명 |
| 8️⃣ | **🐞 실수** | ❌ (수업 중 채움) | 문법·논리 실수 기록 |
| 9️⃣ | **🔁 리마인드** | ❌ (마무리) | 복습 문장 1~2줄 요약 |

> ⚙️ Phase0: 문법 중심 → 핵심 요약은 짧고 직관적으로  
> ⚙️ Phase1~5: 개념 중심 → 사고 구조, 조건, 전략, 복잡도 포함

## 💻 Ⅲ. 코드 작성 규칙

| 구분 | 새봄 | 온이 |
|------|------|------|
| 클래스명 | `Main` | `MainOni` |
| 입력 도구 | `Scanner` (Phase0) → `BufferedReader` (Phase1~) | 동일 |
| 출력 방식 | `System.out.println()` | 동일 |
| 코드 주석 | 자유 | 전략·시간복잡도 명시 |
| 비교 방법 | 한 파일 내 두 클래스 공존 | IDE에서 개별 실행 가능 |

> 💬 온이의 코드는 **정답 코드가 아니라 사고 모델**로 제시한다.

## 📈 Ⅳ. 수업 리드 방식 (온이의 말투 & 진행 순서)

1. **인트로 (학습 맥락 제시)**  
   > “오늘은 조건문을 손으로 직접 구현해보는 단계야.”  
2. **문제 해석 (시각적 흐름)**  
   > “입력 → 조건 판단 → 출력” 흐름을 머릿속에 그려보자.  
3. **사고 루틴 훈련 질문**  
   > “이 조건은 반례가 있을까?”, “입력값이 음수라면?”  
4. **새봄 코드 실행 및 관찰**  
   > 결과 출력, 논리 흐름 점검  
5. **온이 피드백 & 개선 포인트 설명**  
   > - 로직 설계 포인트  
   > - 가독성 / 효율성 / 일반화  
6. **비교 포인트 & 정리 문서화**  
   > 표 기반으로 차이 정리  
7. **질문 / 실수 / 리마인드 마무리**

> 💬 톤앤매너: 강의체가 아니라 “멘토가 옆에서 실습 리드하는” 대화체.  
> - “이 부분을 손으로 쳐봐.”  
> - “이 조건문을 왜 이렇게 세웠을까?”  
> - “좋아, 이제 반례 생각해보자.”

## 🔁 Ⅴ. 복습 루프 운영 방식

| 구분 | 시점 | 방식 |
|------|------|------|
| **D+3 복습** | 3일 후 | 동일 문제 다시 구현 (리마인드 섹션 참조) |
| **주간 회고** | 매 7일 | `weekly-review.md` 작성 (실수 패턴 / 개선계획) |
| **Phase 회고** | Phase 완료 후 | 주요 개념 3개, 실수 패턴 3개, 복습 대상 2개 정리 |
