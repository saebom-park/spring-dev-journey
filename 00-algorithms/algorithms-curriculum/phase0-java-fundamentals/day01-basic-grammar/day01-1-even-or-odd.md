## 💡 핵심 개념 요약

- `%` 연산자: **나머지**를 구할 때 사용.
→ `n % 2 == 0` 이면 짝수, 그렇지 않으면 홀수.
- `if / else`: **조건에 따라 분기하는 제어문.**
- `Scanner`: 표준 입력을 처리하는 가장 기본적인 방법.
- 삼항 연산자 `? :` 를 사용하면 한 줄 표현도 가능.
- 알고리즘에서의 조건문은 단순 분기가 아니라 **사고 구조의 기초.**
→ 나중에 `DFS`, `DP`, `Greedy` 등에서도 조건 판별이 핵심 역할을 함.

---

## 🧾 문제 풀이

### ✏️ 내 풀이

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();

        System.out.println(n % 2 == 0 ? "Even" : "Odd");
    }
}
```

### 💎 온이 풀이

```java
import java.util.Scanner;

public class MainOni {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();

        if (n % 2 == 0) {
            System.out.println("Even");
        } else {
            System.out.println("Odd");
        }
    }
}
// ✅ 전략: 명시적 if/else 구조로 논리 흐름 확인
// ⏱ 시간복잡도: O(1)
// 💬 포인트: 가독성을 높이고 디버깅이 용이한 구조
```

## 🌱 **핵심 정리**

- **핵심 문법:** `%`, `if / else`, 삼항 연산자
- **핵심 구조:** 입력 → 판단 → 출력
- **시간 복잡도:** O(1)
- **학습 포인트:**
    - 조건문은 “논리 흐름의 분기”이자 모든 알고리즘의 뼈대.
    - 삼항 연산자는 조건식 단순화의 첫 단계.
    - 이 패턴은 나중에 `Greedy`, `DP` 조건 업데이트로 이어짐.

## 📒 이해 확장 질문

### 💬 Q1. 음수 입력(`n = -3`)일 때 출력 결과는?

👉 직접 생각해보자.

자바에서 `%` 연산자는 **“나누는 수의 부호가 아니라, 피제수(앞에 오는 수)”의 부호를 따른다.**

즉

```
-3 % 2 == -1
```

결과적으로

```java
if (n % 2 == 0)
```

→ `(-3 % 2 == 0)` ❌ (false)

→ `"Odd"` 출력.

💡 **음수도 홀짝이 있다!** 짝수/홀수는 “부호”가 아니라 **“2로 나누어 떨어지는가”**로 결정된다.

> ✅ 즉, 자바에서는 -3도 홀수로 인식돼.
> 
> 
> (이건 파이썬, C++, 자바 각각 다르게 동작하니까 나중에 비교해볼 예정)
> 

---

### 💬 Q2. `%` 연산자는 어떤 경우에 0이 아닌 값을 반환할까?

- **0이 아닌 나머지**가 남는 경우야.
- 예를 들어 `7 % 3 = 1`, `10 % 4 = 2` 처럼
    
    **나누어떨어지지 않으면 항상 0이 아닌 나머지**가 남지.
    

즉,

> “나머지가 0 → 배수 관계”
> 
> 
> “나머지가 0이 아님 → 배수 아님”
> 

이 원리를 이해하면

→ 나중에 “배수 판별”, “약수 탐색”, “최대공약수(GCD)” 문제를 풀 때 똑같이 써먹을 수 있어.

---

### 💬 Q3. `if (n % 2 == 0)` 대신 `switch`로도 구현할 수 있을까?

이건 문법적으로는 가능해 —

하지만 **`switch`는 정수나 문자열의 ‘정확히 일치하는 값’**을 비교할 때 쓰는 구문이야.

짝수·홀수처럼 “조건(참/거짓)” 판단에는 `if`문이 더 직관적이지.

그래도 예시로 보여주면 이렇게 돼 👇

```java
switch (n % 2) {
    case 0:
        System.out.println("Even");
        break;
    default:
        System.out.println("Odd");
}
```

> ✅ 결론:
> 
> 
> 가능은 하지만, 이런 경우에는 **`if`가 논리적으로 더 자연스럽고 가독성 좋다.**
> 

---

💬 이 섹션에서의 목표는 단 하나야:

> “코드 한 줄을 다른 상황에서도 설명할 수 있는가?”
> 

이걸 반복하다 보면 나중에 새로운 문제를 봐도

자연스럽게 “조건 → 분기 → 결과” 구조가 머릿속에서 떠올라.
