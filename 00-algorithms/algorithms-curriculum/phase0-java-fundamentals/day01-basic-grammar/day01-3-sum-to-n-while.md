## 🧩 문제 설명

정수 `n`이 주어졌을 때,

`while` 문을 이용하여 1부터 `n`까지의 합을 구하는 프로그램을 작성하시오.

**입력 예시**

```
5
```

**출력 예시**

```
15
```

---

## 💡 핵심 개념 요약

- `while` 문은 **조건이 참인 동안 반복**되는 제어문.
→ 반복 횟수가 정해지지 않았거나, 조건에 따라 종료될 때 사용.
- `for` 문은 **초기화, 조건, 증감**이 한 줄에 있지만`while`은 **분리**되어 있어 직접 변수 증가를 관리해야 함.
- 기본 구조:
    
    ```java
    int i = 1;
    while (i <= n) {
        sum += i;
        i++;
    }
    ```
    
- 초기화(`i=1`), 조건(`i<=n`), 증감(`i++`) 세 가지를 모두 신경 써야 함.
- 종료 조건을 잘못 설정하면 **무한 루프** 발생 가능.

---

## 🧾 문제 풀이

### ✏️ 내 풀이

```java
public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        
        int sum = 0;

        while (int i <= n) {
          sum += i;
        }

        System.out.println(sum);
        
    }
}
```

### 💎 온이 풀이

```java
import java.util.Scanner;

public class MainOni {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();

        int sum = 0;
        int i = 1; // 초기화

        // ✅ 핵심: while문을 이용한 누적합
        while (i <= n) {
            sum += i; // 누적
            i++;       // 증감
        }

        System.out.println(sum);
    }
}
// ✅ 전략: 반복 제어 구조를 직접 관리하는 while문
// ⏱ 시간복잡도: O(n)
// 💬 포인트: for문과 달리, 초기화·조건·증감을 분리해 명시적으로 표현
```

### 📌 비교 포인트

| 항목 | 새봄 코드 | 온이 코드 | 차이 |
| --- | --- | --- | --- |
| 변수 선언 | `while (int i <= n)` 내부 선언 | `int i = 1;` 외부 선언 | ✅ 반복 변수는 while 밖에서 선언해야 함 |
| 증감식 | 없음 | `i++` 추가 | ✅ 무한 루프 방지 필수 |
| 조건 | `i <= n` | 동일 | 동일 |
| 누적 | `sum += i` | 동일 | 동일 |
| 출력 | `System.out.println(sum)` | 동일 | 동일 |
| 시간복잡도 | O(n) | O(n) | 동일 |
| 구조 차이 | for문 통합형 | while문 분리형 | 제어 흐름의 명시성 차이 |

> 💬 온이 코멘트
이번 문제의 핵심은 “for문과 while문은 사고 구조는 같지만, 표현의 책임이 다르다”는 점이야.
for는 모든 제어를 한 줄에서 관리하고,
while은 **초기화 / 조건 / 증감**을 각자 명시적으로 관리해야 한다.
그래서 while은 조금 더 “코드를 설계하는 감각”을 기르기 좋지 🌱
> 

---

## 🌱 핵심 정리

- **반복문의 본질:** 조건이 참인 동안 **반복적 누적 수행**
- **for ↔ while 차이:**
    - `for` → 제어(초기화·조건·증감)가 한 줄
    - `while` → 제어가 분리되어 있어 **직접 관리 필요**
- **핵심 구조:**
    
    ```
    초기화 → 조건검사 → 실행 → 증감 → 재검사 → 종료
    ```
    
- **실수 방지 포인트:**
    - 초기화 누락 → `i` undefined 오류
    - 증감식 누락 → 무한 루프
    - 조건 실수 (`<` vs `<=`) → 누락 또는 초과 합산
- **시간 복잡도:** O(n)
- **사고 패턴:**
    - 반복의 대상이 명확하면 for
    - 종료 조건 중심이라면 while

> 💬 **온이 코멘트:**
이번 단계의 목적은 “코드 흐름을 직접 제어하는 감각”을 익히는 거야.
while문은 단순히 for의 대체가 아니라, **조건을 설계하는 사고 훈련 도구**야.
이제 ‘반복 제어’ 자체를 자유롭게 다루게 되면,
나중에 **BFS, DFS, 시뮬레이션 루프** 설계가 훨씬 자연스러워질 거야 🌱
>

## 📒 이해 확장 질문

### 💬 **Q1. for문으로 다시 바꿔 쓸 수 있을까?**

→ ✅ 가능해.

```java
for (int i = 1; i <= n; i++) {
    sum += i;
}
```

사실 while문과 for문은 **논리적으로 동일한 구조**야.

차이는 **제어 변수를 어디서 관리하느냐** 뿐이야.

- for: 제어(초기화·조건·증감)를 한 줄에서 관리
- while: 각 단계를 개발자가 직접 나눠서 제어

---

### 💬 **Q2. i++을 while문 바깥에 두면 어떻게 될까?**

→ ❌ 무한 루프 발생!

증감식이 실행되지 않으면, `i`가 계속 1이어서 `i <= n` 조건이 영원히 참이 된다.

→ 반복문의 “탈출 경로”를 항상 코드 안에 포함해야 해.

---

### 💬 **Q3. 조건을 `i < n`으로 바꾸면?**

→ 마지막 수 n이 빠져서 `1 + 2 + ... + (n-1)`까지만 더해져.

→ 따라서 “1부터 n까지”를 표현할 땐 **`<=`** 가 정확하다.

---

### 💬 **Q4. do-while문으로도 가능할까?**

→ ✅ 가능하다.

단, do-while은 “일단 한 번은 실행하고 조건을 검사”하기 때문에 약간 다르게 작동한다.

```java
int i = 1;
do {
    sum += i;
    i++;
} while (i <= n);
```

→ 차이점: n이 0이라도 한 번은 실행된다는 점!

---

> 💬 **온이 코멘트:**
같은 누적 문제라도 while / for / do-while 세 가지 방식으로 사고해보면
“조건이 코드의 성격을 결정한다”는 걸 명확히 체감할 수 있어.
이게 바로 나중에 **재귀 / 반복 / 종료 조건 설계**로 이어지는 핵심 감각이야 🌱
>

## 🐞 실수 체크

| 구분 | 실수 내용 | 원인 | 교정 방법 |
| --- | --- | --- | --- |
| 변수 선언 | `while (int i <= n)` 내부 선언 | while 문 안에서 타입 선언 불가 | 반복 변수는 while **밖에서 선언** (`int i = 1;`) |
| 증감 누락 | `i++` 빠짐 | 증감식 생략 시 조건 변화 없음 | 루프 끝마다 반드시 `i++` 수행 |
| 조건식 실수 | `i < n` | 등호 빠져 마지막 값 누락 | “1부터 n까지”면 항상 `<=` 사용 |
| 무한 루프 | 종료 조건 미달성 | i가 계속 증가하지 않거나 조건이 잘못됨 | while문 실행 전후 i 변화 추적 |
| 변수 재사용 | n, i 혼동 | 입력 변수와 제어 변수 혼동 | 역할 분리: n=입력, i=루프, sum=누적 |

> 💬 **온이 코멘트:**
while문에서의 실수는 대부분 “조건이 변화하지 않는 상태”에서 발생해.
코드보다 먼저 **‘탈출 조건이 존재하는가?’** 를 머릿속에서 체크하는 습관을 들이자 🌱
>

## 🔁 리마인드

> “while문은 흐름 제어의 모든 책임을 개발자에게 맡기는 반복문이다.”
> 
- for문은 자동으로 제어를 관리하지만, while문은 **직접 초기화·조건·증감을 설계**해야 한다.
- 이 감각이 익숙해지면 나중에 **DFS, BFS, 시뮬레이션, 입력 루프 처리**를 완벽히 제어할 수 있게 된다.
- 루프를 작성하기 전, 반드시 스스로에게 질문하자:
    
    **“이 반복은 어디서 시작해서, 언제 끝나는가?”**
