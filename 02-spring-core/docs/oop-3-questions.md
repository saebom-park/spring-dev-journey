> (코드: `Vehicle.java`, `SmartWasher.java` 기준)

---

| 질문 | 답변 요약 |
| --- | --- |
| ElectricBike만 생성했는데 왜 부모 생성자도 같이 실행돼? | 자식 객체 만들면, 자바는 부모 생성자부터 자동으로 호출해. 상속받은 필드랑 기능들을 초기화해줘야 하니까, 부모 → 자식 순서로 생성자 호출됨! |
| `super()` 안 썼는데도 부모 생성자가 실행되던데? | 자바가 기본 생성자일 경우 `super()`를 자동으로 삽입해줘. 그래서 명시 안 해도 호출되는 거야. 근데 부모 생성자에 인자가 있으면 반드시 `super(...)`를 직접 써야 해. |
| 부모 객체를 따로 생성하는 경우도 있던데, 그건 왜 그래? | 자식 객체만으로 충분할 때도 있지만, 상황에 따라 **부모 클래스만 쓰고 싶을 때**가 있어. 예를 들면, Vehicle 목록처럼 **공통 기능만 필요할 때**는 부모 객체만 따로 만들어도 돼. |
| 부모를 abstract로 만들면 어떻게 돼? | 추상 클래스는 객체로 못 만들고, 상속 전용으로만 써. 부모는 설계 틀만 주고, 진짜 구현은 자식 클래스가 하는 구조야. 실무에서 되게 자주 나와! |
| `protected`는 뭐야? | `protected`는 **같은 패키지 + 자식 클래스**에서는 접근 가능한 접근제어자야. `private`처럼 완전 잠그진 않고, 상속 구조에선 내부 구현을 공유할 수 있게 열어줘. |
| `public`, `private`, `protected`, `default` 차이점 좀 알려줘 | ✔️ `public`: 어디서든 접근 가능✔️ `private`: 같은 클래스 안에서만 접근 가능✔️ `default` (생략): 같은 패키지 안에서만 가능✔️ `protected`: 같은 패키지 + 상속 관계면 접근 가능 |
