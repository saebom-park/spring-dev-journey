# [OOP-5단계] 질문노트: 다형성과 형변환 (ShapeExample.java)

> 💬 봄이가 직접 궁금해서 물어본 것들 + 온이의 설명 요약  
> (코드: ShapeExample.java 실습 기준)

---

### 💡 1. 그냥 `Admin admin = new Admin();` 하면 되는데 왜 `Member m = new Admin();`을 쓰는 거야?

| 질문 | 답변 요약 |
|------|-----------|
| 굳이 부모 타입으로 참조하는 이유가 뭐야? | `Admin`만 쓸 거면 직접 생성해도 돼. 하지만 실무에서는 어떤 객체가 들어올지 모르기 때문에 **상위 타입(Member)으로 통합해서 다형성으로 관리**하는 게 일반적이야. 예: `Member[]` 배열에 여러 타입의 자식 객체 담기 가능. |

---

### 💡 2. 다운캐스팅할 땐 왜 `instanceof`도 같이 써야 해?

| 질문 | 답변 요약 |
|------|-----------|
| 그냥 캐스팅하면 안 돼? | 다운캐스팅은 **실제 객체가 자식 타입이 아닐 경우 `ClassCastException` 예외가 발생**해. `instanceof`는 타입 확인용 안전장치 역할을 해. 실무에서는 반드시 같이 써! |

---

### 💡 3. 부모 타입으로 자식 객체를 참조하면 오버라이딩된 메서드도 자식 기준으로 실행돼?

| 질문 | 답변 요약 |
|------|-----------|
| 변수는 부모인데 자식 메서드가 실행된다고? | 맞아! **실제 인스턴스가 자식이면 자식 메서드가 우선 실행**돼. 이걸 **동적 바인딩(dynamic binding)** 이라고 해. 자바는 실행 시점 기준으로 메서드를 결정해. |

---

### 💡 4. 다운캐스팅하면 자식 메서드도 전부 쓸 수 있어?

| 질문 | 답변 요약 |
|------|-----------|
| 부모로는 안 됐던 메서드가 갑자기 가능해지는 거야? | ✅ 응! 형변환만 잘 하면 자식 클래스의 고유 메서드까지 전부 쓸 수 있어. 하지만 **형변환 없이 자식 메서드에 접근하면 오류 발생**하니까 항상 타입 확인 필수! |

---

### 🌱 정리 키워드

- 다형성 → 부모 타입으로 다양한 자식 객체 관리 가능
- `Member m = new Admin()` → 실무에서 흔한 다형성 활용 방식
- `instanceof` → 다운캐스팅 전 안전 확인
- 동적 바인딩 → 실제 객체 기준으로 메서드 실행
- 다운캐스팅 → 자식 메서드까지 접근 가능하지만 주의 필요
