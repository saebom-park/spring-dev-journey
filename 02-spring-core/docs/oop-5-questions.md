# [OOP-5단계] 질문노트: 다형성과 형변환 (ShapeExample.java)

> 💬 봄이가 실습 중 궁금했던 개념들을 모아서 정리한 질문노트
> 

| 질문 | 답변 요약 |
| --- | --- |
| 그냥 `Admin admin = new Admin();` 하면 되는데, 왜 `Member m = new Admin();` 하고 다운캐스팅해야 해? | `Admin`만 쓸 거면 그 방식도 OK. 하지만 실무에서는 **어떤 객체가 들어올지 모를 때**가 많아. 그래서 `Member`라는 상위 타입으로 통합해두고, 필요할 때만 자식 클래스로 꺼내 쓰는 거야. 다형성 덕분에 `Member[]` 배열로 `Admin`, `Customer`, `Guest` 같은 다양한 객체들을 한 번에 관리할 수 있어. |
| 그럼 다운캐스팅은 왜 꼭 `instanceof`랑 같이 써야 해? | 다운캐스팅은 **실제 객체가 자식 타입이 아닐 경우 에러가 나기 때문**이야. `instanceof`는 "이 객체 진짜 Admin 맞아?"라고 물어보는 안전장치야. 안 쓰면 실행 중에 `ClassCastException`이 터져서 위험해. |
| 부모 타입 변수로 자식 객체 참조하면 오버라이딩된 메서드도 자식 기준으로 실행돼? | 맞아! 그게 바로 다형성의 핵심. 변수 타입이 부모더라도, **실제 인스턴스가 자식이면 자식의 메서드가 우선 실행**돼. 이를 동적 바인딩(dynamic binding)이라고 해. |
| 다운캐스팅하면 자식 메서드는 다 쓸 수 있어? | ✅ 응! 형 변환만 잘 해주면 자식 클래스에만 있는 메서드까지 전부 호출 가능해. 하지만 **절대 조심해야 할 건: 부모 타입에 있는 메서드는 무조건 호출 가능하지만, 자식은 다운캐스팅 없이는 절대 못 써!** |
