# [OOP-6단계] 질문노트: 추상 클래스와 강제 구현 (DeviceExample.java)

> 💬 추상 클래스 실습 중 봄이가 궁금했던 핵심 개념 정리
> 

| 질문 | 답변 요약 |
| --- | --- |
| 추상 클래스는 왜 직접 인스턴스를 만들 수 없지? | 추상 클래스는 **설계용 클래스**이기 때문이야. 내부에 **구현되지 않은 추상 메서드**가 있을 수 있어서, 직접 객체를 만들 수 없어. 자식 클래스가 상속해서 구체적인 동작을 정의해야만 사용 가능해. |
| 추상 메서드는 꼭 자식 클래스에서 오버라이딩해야 해? | ✅ 반드시 해야 해. 추상 메서드는 본문이 없는 "설계만 존재하는 메서드"라서, 자식 클래스가 **그 기능을 반드시 구현하도록 강제**하는 구조야. |
| 추상 클래스도 다형성으로 사용할 수 있어? | 가능해! 예를 들어 `Phone[]` 배열로 `SmartPhone`, `FeaturePhone` 객체들을 담고 다형성으로 관리할 수 있어. 단, **추상 클래스 자체로는 객체 생성 불가**이지만, **부모 타입으로는 동작 가능**해. |
