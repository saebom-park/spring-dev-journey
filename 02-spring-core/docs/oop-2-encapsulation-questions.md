### 💡 1. duplicate class: Person 오류가 뭐야?

| 질문 | 답변 요약 |
| --- | --- |
| 패키지가 다르더라도 왜 오류가 나는 거야? | 자바 프로젝트 내에서 **패키지 선언 없이 동일한 클래스명이 존재**하면 IntelliJ는 컴파일 시 충돌로 인식함 |
| 어떻게 해결해? | `package`를 명시하고, 파일을 **해당 패키지 경로에 정확히 이동**해야 해결됨 (`src/oop/Student.java`) |

### 💡 2. final 필드 추천 경고는 무슨 의미야?

| 질문 | 답변 요약 |
| --- | --- |
| 왜 IntelliJ가 final 추천하는 거야? | 생성자에서만 값을 설정하고 이후 변경하지 않는 필드는 `final`로 선언하는 게 **더 안전하고 명확**함 |
| 꼭 바꿔야 해? | ❌ 강제는 아님. 하지만 **불변성을 명시하고 싶을 때는 바꿔주는 것이 좋음** |

---

### 💡 3. 절대값 구하는 법은 어떻게 해?

| 질문 | 답변 요약 |
| --- | --- |
| 어떤 메서드 써야 해? | `Math.abs()` 사용하면 정수든 실수든 절대값 반환됨 |
| 예시 코드? | `int a = Math.abs(-5);` → `a = 5` / `double b = Math.abs(-3.14);` → `b = 3.14` |

---

### 💡 4. 배열에 값 넣는 문법이 궁금해!

| 질문 | 답변 요약 |
| --- | --- |
| 자바에서 배열 요소에 값 넣는 방법은? | `배열[인덱스] = 값;` 형식으로 직접 대입하면 됨 |
| 예시 코드? | `int[] arr = new int[2];
arr[0] = 10;
arr[1] = 20;` |
| 문자열 배열도 똑같아? | 똑같이 가능: `String[] names = new String[2]; names[0] = "봄이";` |

---

### 🌱 정리 키워드

- `duplicate class` → 같은 이름 클래스는 충돌 가능
- `package`와 파일 경로는 1:1 정확히 맞춰야 함
- 생성자에서만 초기화된 필드는 `final` 가능성 있음
- `Math.abs()` → 절대값 계산 함수
- 배열 값 대입 → `arr[index] = value`
