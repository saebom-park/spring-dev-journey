### 💡 1. 왜 숫자에 L을 붙여야 해?

| 질문 | 답변 요약 |
| --- | --- |
| `Long id = 123L;`처럼 L을 붙이는 이유는 뭐야? 그냥 123 쓰면 안 돼? | 자바는 정수 리터럴을 기본적으로 `int`로 해석하기 때문에, `Long` 타입으로 명시하려면 반드시 `L`을 붙여야 해. 특히 값이 커질 경우 `int` 범위를 초과하므로 `L`을 붙이는 습관이 중요해. |

---

### 💡 2. 왜 값이 작아도 Long을 써야 해?

| 질문 | 답변 요약 |
| --- | --- |
| 123처럼 작은 값인데 굳이 Long을 써야 해? int로 하면 안 돼? | 값이 작더라도, 해당 변수가 `Long` 타입이라면 L을 붙이는 게 문법적으로 명확하고 실무적으로 안정적이야. 나중에 큰 값을 다뤄야 할 때도 일관성이 있고, 불필요한 형변환을 피할 수 있어. |

---

### 💡 3. 콘솔에 입력할 때도 L 붙여야 해?

| 질문 | 답변 요약 |
| --- | --- |
| 콘솔 입력으로 123을 넣을 때도 123L로 써야 해? | ❌ 절대 아냐! 콘솔 입력은 문자열로 받아들이기 때문에 `123L`처럼 입력하면 에러나. 콘솔에는 그냥 `123`이라고 입력하면 되고, 코드 내부에서 `Long`으로 파싱하면 돼. |

---

### 💡 4. 왜 실행 중에 non-zero exit value 1 오류가 나?

| 질문 | 답변 요약 |
| --- | --- |
| `Process ... finished with non-zero exit value 1`은 무슨 뜻이야? | 자바 프로그램 실행 중 예외가 발생해서 비정상적으로 종료됐다는 뜻이야. 보통 `Scanner` 입력 실수나 Bean 주입 오류 같은 게 원인일 수 있어. 가장 위쪽 로그에서 `Caused by:`를 확인해야 진짜 원인을 알 수 있어. |

---

### 🌱 정리 키워드

- L 접미사 = `int`가 아닌 `Long` 타입임을 명시하기 위한 식별자
- 콘솔 입력값 = 문자열 → 숫자형으로 파싱해야 함 (`Long.parseLong()` 등)
- 타입 일치 = 변수 타입과 값 타입을 맞추는 습관 필수
- 런타임 오류 파악 = `non-zero exit` 에러는 예외가 발생했다는 신호
- 로그 분석 = `Caused by:`를 가장 먼저 확인하자
