# 🧷 02. 객체지향 설계 & Spring 기초

> 📁 폴더 위치: `02-spring-core/`
>
> 자바 객체지향(OOP) 개념을 기반으로, 스프링의 IoC/DI, 계층 구조, 부트 프로젝트 구조, REST 컨트롤러, DTO 처리까지 실습 중심으로 익히는 챕터입니다. 백엔드의 기초 설계 역량과 스프링 기반 구현 감각을 함께 키우는 것이 목표입니다.

---

## ✅ 학습 목표

- 자바 기반 OOP 핵심 개념 (캡슐화, 상속, 다형성 등) 체득
- 객체 설계를 통해 협력 관계와 역할 분리 감각 익히기
- 스프링의 IoC/DI 개념을 실습으로 습득
- 3계층 구조(Controller-Service-Repository) 설계 패턴 경험
- Spring Boot 프로젝트 구조와 자동 구성 이해
- @RestController, 요청 매핑, DTO 직렬화 흐름 정복

---

## 🧠 전체 학습 구성 (총 13단계)

| 단계       | 주제            | 주요 개념 키워드                            |
|----------|---------------|--------------------------------------|
| OOP-1    | 객체 vs 클래스     | 인스턴스, `class`, `new`, `.`            |
| OOP-2    | 캡슐화           | `private`, `getter/setter`, 정보 은닉    |
| OOP-3    | 상속 구조         | `extends`, 부모/자식 클래스                 |
| OOP-4    | 오버라이딩         | `@Override`, 재정의                     |
| OOP-5    | 다형성과 형변환      | 업캐스팅, 다운캐스팅, 동적 바인딩                  |
| OOP-6    | 추상화 설계        | `abstract`, 강제 구현, 템플릿 메서드           |
| OOP-7    | 인터페이스 설계      | `interface`, `implements`, 역할 중심     |
| OOP-8    | 객체 협력 관계      | 메시지 전달, 의존성, 역할 분리                   |
| SPRING-1 | IoC, DI 개념    | 스프링 컨테이너, 의존성 주입                     |
| SPRING-2 | 3계층 구조        | Controller → Service → Repository    |
| SPRING-3 | 부트 프로젝트 구조    | 컴포넌트 스캔, 패키지 구조                      |
| SPRING-4 | 요청 매핑 흐름      | `@RestController`, `@RequestMapping` |
| SPRING-5 | DTO & JSON 매핑 | Request/Response DTO, 직렬화            |
