# 02-spring-core
Java 객체지향 개념을 기반으로, 스프링의 IoC/DI, 계층 구조, REST 컨트롤러, DTO 처리까지 실습 중심으로 학습하는 단계입니다.  
백엔드의 기초 설계 역량과 스프링 기반 구현 감각을 함께 구축합니다.

---

## 🗂️ 구성 안내
- `docs/` : OOP 및 Spring 개념 요약, 예시 코드, 실습 미션
- `src/`  : 손코딩 실습 코드 모음

---

## 🎯 학습 목표
- 자바 기반 객체지향 핵심 개념(캡슐화, 상속, 다형성 등)을 코드로 체득한다.
- 객체 간 협력 구조를 설계하며 역할 분리 감각을 익힌다.
- Spring IoC/DI 원리를 이해하고 의존성 주입 방식을 실습한다.
- Controller → Service → Repository 로 이어지는 3계층 구조를 직접 구현한다.
- Spring Boot 프로젝트 구조와 컴포넌트 스캔 흐름을 이해한다.
- REST 컨트롤러, 요청 매핑, DTO 직렬화까지 실무 기본기를 쌓는다.

---

## 🧠 전체 학습 구성 (총 13단계)

| 단계       | 주제            | 주요 개념 키워드                            |
|----------|---------------|--------------------------------------|
| OOP-1    | 객체 vs 클래스     | 인스턴스, `class`, `new`, `.`            |
| OOP-2    | 캡슐화           | `private`, getter/setter             |
| OOP-3    | 상속 구조         | `extends`, 부모/자식 관계                  |
| OOP-4    | 오버라이딩         | `@Override`, 재정의                     |
| OOP-5    | 다형성 & 형변환     | 업캐스팅, 다운캐스팅, 동적 바인딩                  |
| OOP-6    | 추상화 설계        | `abstract`, 강제 구현                    |
| OOP-7    | 인터페이스 설계      | `interface`, 역할 중심 설계                |
| OOP-78   | 객체 협력 관계      | 메시지 전달, 의존성, 역할 분리                   |
| SPRING-1 | IoC, DI 개념    | 스프링 컨테이너, 의존성 주입                     |
| SPRING-2 | 3계층 구조        | Controller → Service → Repository    |
| SPRING-3 | 부트 프로젝트 구조    | 패키지 구조, 컴포넌트 스캔                      |
| SPRING-4 | 요청 매핑 흐름      | `@RestController`, `@RequestMapping` |
| SPRING-5 | DTO & JSON 매핑 | Request/Response DTO, 직렬화            |

## 🧭 학습 흐름

1. 각 단계의 `.md` 문서에서 주요 개념을 먼저 정리한다.
2. 제공된 예시 코드를 직접 구조적으로 따라 치며 실습한다.
3. 코드 흐름이 복잡하거나 설명이 필요한 경우 `-details.md` 문서에서 전체 실행 흐름을 확인한다.
4. 실습 중 발생한 오류는 `-mistakes.md`에 기록한다.
5. 학습 중 떠오른 질문은 `-questions.md`에 Q/A 형식으로 정리한다.
6. OOP 개념과 Spring 구조를 연결하며 “설계 → 구현 → 흐름”을 함께 확인하는 것을 목표로 한다.

## 📌 문서 종류별 기록 원칙

### **`-details.md`**
- 예시 코드의 **전체 구조와 실행 흐름**을 단계별로 해설
- 클래스 역할, 어노테이션 동작, 요청 처리 흐름을 구조적으로 설명
- “코드 → 설명 → 흐름 요약 → 개념 포인트” 순으로 작성
- 대상 예시(실습 코드)에 100% 맞춰 작성하는 것이 원칙

### **`-questions.md`**
- 학습 중 나온 질문을 Q/A 형식으로 정리
- 답변은 핵심만 요약하며 실무 관점을 우선 반영

### **`-mistakes.md`**
- 실습 중 발생한 실수를 실제 코드와 함께 정리
- 잘못된 코드 → 수정 코드 → 원인 → 요약 순으로 정리
- 반복 방지를 위한 오류 패턴 중심 구성
