# ✅ 할일 관리 시스템 (todo-app)

> 개인 할일을 체계적으로 관리할 수 있는  
> **상태 변경과 카테고리 기반의 Todo 관리 서비스**입니다.  
> 우선순위, 일정, 반복 설정까지 포함한 실무 수준의 기능을 제공합니다.

---

## 💡 시나리오

> 사용자는 할일을 등록하고 카테고리별로 분류할 수 있으며,  
> 우선순위와 마감일을 설정하여 체계적으로 관리할 수 있습니다.  
> 할일의 상태를 변경하며 진행상황을 추적하고, 반복 일정도 설정할 수 있습니다.

---

## 📋 요구사항

### 1. 할일 기본 기능
- 할일 등록 / 수정 / 삭제
- 할일 목록 조회 (카테고리별, 상태별 필터링)
- 상태 변경 (대기 → 진행 → 완료)
- 우선순위 설정 (높음/보통/낮음)

### 2. 카테고리 관리
- 카테고리 생성 / 수정 / 삭제
- 색상별 카테고리 구분
- 카테고리별 할일 필터링

### 3. 일정 관리
- 시작일 / 마감일 설정
- 완료일 자동 기록
- 마감일 기준 정렬

### 4. 반복 설정 (고급)
- 반복 할일 등록
- 반복 시작일 / 종료일 설정
- 반복 간격 설정

## 🔄 개발 단계

### 1단계: 기본 CRUD
- User, Todo 엔티티 + 기본 API

### 2단계: 상태 관리
- 상태 enum 추가 + 상태 변경 API

### 3단계: 카테고리 + 우선순위
- Category 엔티티 + 필터링 기능

### 4단계: 일정 관리
- Schedule 엔티티 + 마감일 기능

### 5단계: 반복 설정
- RepeatSetting 엔티티 + 반복 로직

## 🧪 실행 방법 (작성 예정)

1. 백엔드: `./backend` 디렉토리에서 `./gradlew bootRun`
2. 프론트엔드: `./frontend` 디렉토리에서 `npm install` → `npm run dev`
