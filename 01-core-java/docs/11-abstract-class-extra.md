# [11단계] 추가 개념 정리

### 💡 핵심 개념 요약

| 주제 | 설명 |
| --- | --- |
| 기본 생성자 | 파라미터가 없는 생성자. 자바가 자동 생성하지만, **직접 생성자를 작성하면 생략됨** |
| super() 호출 | 자식 클래스 생성자에서 부모 생성자 호출할 때 사용. 부모에 기본 생성자가 없으면 오류 발생 |
| 후속 필드 설정 | 기본 생성자를 통해 객체 생성 후, `.name`, `.age` 등 필드를 나중에 설정 가능 |
| 프레임워크 필수 | Spring, JPA, Jackson 등은 내부 동작을 위해 **기본 생성자 필수 요구** |
| 다형성 | 부모 타입으로 여러 자식 객체를 다룰 수 있는 객체지향의 핵심 원리 |

---

### 🧾 예시 코드 1 — 기본 생성자 자동 생성 ❌

```java
class Animal {
    Animal(String name) {
        this.name = name;
    }
}

class Dog extends Animal {
    Dog() {
        super(); // ❌ 오류! 기본 생성자가 없음
    }
}
```

---

### 🧾 예시 코드 2 — 기본 생성자 직접 작성 ✔️

```java
class Animal {
    Animal() {} // ✅ 기본 생성자

    Animal(String name) {
        this.name = name;
    }
}
```

이렇게 하면 `super();` 또는 `new Animal();` 모두 사용 가능해!

---

### 🧾 예시 코드 3 — 다형성 (Polymorphism)

```java
Animal a1 = new Dog("도비");
Animal a2 = new Cat("나비");

a1.speak(); // 멍멍!
a2.speak(); // 냥냥~
```

---

### 📌 포인트 요약

- **기본 생성자**: 자바가 자동 생성하지만, 개발자가 생성자를 하나라도 만들면 자동 생성 ❌
- **super()**: 부모 생성자를 호출할 때 사용. 기본 생성자 없으면 오류 발생
- **다형성**: 부모 타입 하나로 여러 자식 객체를 다룰 수 있음 (`Animal a = new Dog();`)
- 다형성을 활용하면 **배열, 컬렉션, DI 설계 등 유연한 코드 구조 가능!**
