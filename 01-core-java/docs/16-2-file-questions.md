### 💡 1. `isFile()`과 `isDirectory()`는 왜 필요해요?

| 질문 | 답변 요약 |
| --- | --- |
| 파일 이름에 `.txt`가 있으면 파일 아닌가요? | 아닐 수도 있어요! 단지 **이름이 .txt일 뿐**, 실제로는 폴더일 수도 있음 |
| 그러면 `isFile()`은 뭘 검사하는 건가요? | 운영체제가 판단한 **진짜 파일인지** 여부 (`FileSystem` metadata 검사) |
| 결론은? | **이름을 믿지 말고**, `isFile()`과 `isDirectory()`로 확실하게 확인해야 해요! |

### 💡 2. 경로는 맞는데 파일이 존재하지 않는다고 나와요...

| 질문 | 답변 요약 |
| --- | --- |
| `SampleFile.txt`는 분명히 `src/` 안에 있는데 왜 못 찾아요? | 자바는 **프로젝트 루트 디렉토리 기준 상대경로**로 찾기 때문에, `src/`까지 경로를 적어줘야 해요 |
| 그럼 루트는 어디예요? | `System.getProperty("user.dir")`로 확인하면 현재 실행 디렉토리가 나와요 |
| 해결 방법은? | 상대 경로를 `new File("01-core-java/src/SampleFile.txt")`처럼 정확히 지정하거나, 절대 경로를 사용하세요 |

### 💡 3. 확장자 안 쓰면 왜 못 찾아요?

| 질문 | 답변 요약 |
| --- | --- |
| 입력한 경로가 `SampleFile`인데 왜 안 돼요? | 자바는 **문자열 그대로 경로 해석**을 하기 때문에, `SampleFile.txt`가 아니라면 못 찾는 거예요 |
| 해결 방법은? | 입력 시 `.txt` 확장자를 포함하거나, 코드에서 `.txt` 자동 추가해주기 (`if (!path.endsWith(".txt")) { ... }`) |

---

### 🌱 정리 키워드

- 파일 경로는 항상 실행 디렉토리 기준으로 작동
- 확장자는 **이름일 뿐**이고, 파일 여부는 `isFile()`로 확인
- `System.getProperty("user.dir")` → 디버깅 필수 도구!
