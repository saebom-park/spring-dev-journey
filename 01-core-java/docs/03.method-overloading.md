# [3단계] 메서드 오버로딩

> 같은 이름의 메서드를 매개변수만 다르게 해서 여러 개 정의할 수 있어요!
> 
> 
> 같은 동작이지만 **입력 형태가 다른 경우**, 오버로딩이 진짜 편해요 😎
> 

---

### 💡 핵심 개념 요약

| 개념 | 설명 |
| --- | --- |
| 오버로딩 | 같은 이름의 메서드를 매개변수만 다르게 정의 |
| 매개변수 차이 | 개수, 타입, 순서가 다르면 OK |
| 리턴 타입은 상관 없음 | 오버로딩 구분 기준은 오직 "매개변수" |
| 코드 재사용 | 비슷한 동작을 하나의 이름으로 묶어서 사용 가능 |

---

### 🧾 예시 코드

```java
public class Dog {
    String name;
    int age;

    Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void introduce() {
        System.out.println("이름: " + name + ", 나이: " + age);
    }

    void introduce(String mood) {
        System.out.println("이름: " + name + ", 기분: " + mood);
    }
}

public class Main {
    public static void main(String[] args) {
        Dog aboo = new Dog("아부", 7);
        aboo.introduce();              // 이름: 아부, 나이: 7
        aboo.introduce("기분 좋아!");   // 이름: 아부, 기분: 기분 좋아!
    }
}
```

---

### 📌 포인트 요약

- 오버로딩은 같은 이름의 메서드를 **여러 개 정의 가능**
- 구분 기준은 반드시 **매개변수**! (개수/타입/순서)
- 리턴 타입이 다르기만 하면? → ❌ 에러!
- **코드 명확성 + 유연성 향상**에 좋음

---

### 🧪 실습 미션

> ✅ Book 클래스에 showInfo()를 오버로딩
> 
> 
> 기본 출력 + 감정 상태를 포함한 출력 메서드 만들어보자!
> 

```java
Book b = new Book("불변의 법칙", "봄이잇");
b.showInfo();
// 출력: 제목: 불변의 법칙, 저자: 봄이잇

b.showInfo("기분 최고!");
// 출력: 제목: 불변의 법칙, 저자: 봄이잇, 감정: 기분 최고!
```
