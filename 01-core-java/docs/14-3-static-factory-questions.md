### 💡 1. 생성자와 static 메서드 구조

| 질문 | 답변 요약 |
| --- | --- |
| `public static FactoryMember create(String name)`이 생소해… create는 왜 붙는 거야? | `create`는 단순히 static 메서드 이름! 객체를 생성하는 메서드일 땐 관례적으로 create/from/of 등을 자주 씀 |
| FactoryMember이 메서드 이름인 줄 알았는데, 클래스 이름이더라? | 맞아! 메서드 앞에 오는 건 "리턴 타입"이야. 즉, 이 메서드는 FactoryMember 객체를 리턴한다는 뜻! |
| `FactoryMember m = new FactoryMember();` 에 왜 name을 안 넣지? | 이 경우에는 기본 생성자를 쓰기 때문이야! 값을 나중에 m.name = name; 으로 넣고 있어 |
| 그럼 아예 생성자에서 name을 받으면 안돼? | 돼! 그게 더 안전하고 실무적이야. 그래서 "매개변수 생성자" 버전도 만들어본 거야 |
| static 메서드는 왜 FactoryMember 안에서 정의 안 하고, 다른 MemberFactory 같은 클래스에서 정의해? | static 메서드는 꼭 같은 클래스에 있어야 하는 건 아니야. 외부에서 만들어주는 구조는 "정적 팩토리 메서드" 패턴이라고 해 |

### 💡 2. 클래스 전체를 private으로 만들면 안 돼?

| 질문 | 답변 요약 |
| --- | --- |
| 클래스 정의할 때 `private class MemberFactory {}` 이렇게 하면 생성자 안 막아도 되지 않아? | ❌ 클래스 자체를 private으로 만들면 **외부에서 아예 접근 불가**라서 객체 생성뿐 아니라 메서드도 호출 못 해. 그래서 보통 생성자만 private으로 막아! |

### 💡 3. static 블록 사용 이유

| 질문 | 답변 요약 |
| --- | --- |
| static 블록으로 role을 설정하던데, 왜 메서드에서 안 하고 굳이 static 블록 써? | static 블록은 클래스가 **처음 로딩될 때 한 번만 실행**돼. static 필드를 초기화할 때 자주 써 |

### 💡 4. 기본 생성자 vs 매개변수 생성자

| 질문 | 답변 요약 |
| --- | --- |
| 기본 생성자와 매개변수 생성자의 차이는? | 기본 생성자는 값을 나중에 설정해야 해. 매개변수 생성자는 객체 생성과 동시에 값 설정을 강제할 수 있어서 실무에서 선호돼 |
| 매개변수 생성자가 있으면 기본 생성자는 자동으로 만들어져? | ❌ 아니! 매개변수 생성자를 정의하면, 기본 생성자는 **자동으로 만들어지지 않아**. 필요하면 명시적으로 써야 해 |

### 🌱 정리 키워드

- `static factory method`는 객체 생성을 책임지는 정적 메서드
- `create()`는 관례적으로 자주 쓰는 이름일 뿐, 자유롭게 정할 수 있음
- `private 생성자`는 외부에서 객체 생성을 막고, static 메서드만 허용함
- 클래스 전체를 `private class`로 만들면 외부에서 접근 자체가 불가능
- `static {}` 블록은 클래스가 처음 로드될 때 한 번 실행됨
- **기본 생성자 vs 매개변수 생성자**: 값 설정 시점과 안정성의 차이
