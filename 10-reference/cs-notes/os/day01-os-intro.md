# Day01. 운영체제 개요 & 역할

## 1. 주제 정의

운영체제(Operating System, OS)는 컴퓨터 하드웨어와 응용 프로그램
사이에서 자원을 관리하고 사용자에게 인터페이스를 제공하는
소프트웨어이다.

## 2. 하위 구분

-   GUI (Graphical User Interface): 아이콘, 창, 버튼 등 시각적 요소 기반
    인터페이스 (예: Windows, macOS)
-   CUI (Character User Interface): 텍스트 기반 명령어 인터페이스 (예:
    Linux, DOS)

## 3. 주요 역할

-   프로세스 관리: 실행 중인 프로그램의 생성, 스케줄링, 종료 제어
-   메모리 관리: 프로그램이 사용할 메모리 공간을 할당/해제
-   파일 관리: 파일의 생성, 삭제, 읽기/쓰기 수행
-   입출력 관리: 디스크, 키보드, 마우스 등 장치 접근 제어
-   보안 관리: 사용자 계정과 권한 제어, 시스템 보호

## 4. 구조

-   커널(Kernel): 운영체제의 핵심. 하드웨어와 직접 상호작용하며
    프로세스, 메모리, I/O 제어
-   시스템 콜(System Call): 응용 프로그램이 커널 기능을 사용할 수 있도록
    제공하는 인터페이스
-   쉘(Shell): 사용자가 입력한 명령어를 해석해 커널에 전달

## 5. 면접 예상 질문

-   Q: 운영체제가 무엇인가요?\
    A: 운영체제는 컴퓨터 하드웨어와 응용프로그램 사이에서 자원을
    관리하고 사용자와 시스템 간 인터페이스를 제공하는 소프트웨어입니다.

-   Q: GUI와 CUI의 차이는 무엇인가요?\
    A: GUI는 그래픽 요소로 조작하는 환경이고, CUI는 명령어를 직접
    입력하는 환경입니다.

-   Q: 커널과 시스템 콜의 관계를 설명해보세요.\
    A: 커널은 운영체제의 핵심 부분이고, 시스템 콜은 응용 프로그램이 커널
    기능을 사용할 수 있게 해주는 인터페이스입니다.

## 6. 핵심 요약

-   운영체제 = 자원 관리 + 사용자 인터페이스 제공
-   다섯 가지 역할 = 프로세스 / 메모리 / 파일 / 입출력 / 보안
-   구조 = 커널(핵심), 시스템 콜(연결), 쉘(명령 해석)
