# 10. CS 정리 노트 (cs-notes)
백엔드 엔지니어로 성장하기 위해 필수적인 **CS 전 영역을 체계화하여 정리한 문서 저장소**입니다.  
운영체제, 네트워크, 데이터베이스, 디자인 패턴 등 핵심 기초지식을  
**Day 단위로 구조화**해 개념을 정확하게 이해하고 실무에 적용할 수 있도록 구성했습니다.

이 저장소는 단순 암기가 아닌,  
**원리 기반 이해 → 구조화 → 기술 언어로 설명 → 실무 적용**을 목표로 합니다.

---

# 🎯 프로젝트 목표

- 백엔드 개발에 필요한 **핵심 CS 지식의 체계적 정리**  
- 개념 · 구조 · 동작 원리에 대한 **정확한 기술적 이해**  
- 실제 시스템 구현 또는 트러블슈팅에 연결되는 **실무 관점 기반의 설명**  
- 장기적으로 참고 가능한 **지속 가능한 CS 아카이브 구축**

---

# 📂 폴더 구조

```
/cs
 ├── os/               # 운영체제 지식
 ├── network/          # 네트워크 프로토콜·통신 구조
 ├── database/         # 데이터베이스 내부 동작 및 트랜잭션 구조
 ├── dev-basic/        # 개발자 필수 기본기
 ├── design-pattern/   # 객체지향 설계 및 GoF 디자인패턴
 ├── docs/       # 전체 커리큘럼 (v1, v2) 및 문서 규칙
 └── README.md
```

각 주제 폴더 안에는 Day 단위 개념 문서와 응용 문서가 포함됩니다.

---

# 📘 문서 구성 방식

## 1) 개념 문서 (concept)  
`DayNN-주제-concept.md`

- 개념 정의  
- 구성 요소와 구조  
- 동작 원리  
- 내부 메커니즘  
- 실무에서의 활용 포인트  
- 오해 정리  
- 마지막에 **핵심 키워드 요약**

→ 읽기만 해도 기술의 본질을 명확하게 이해할 수 있도록 구성된 “교재형 문서”

---

## 2) 응용 문서 (questions)  
`DayNN-주제-questions.md`

- 핵심 개념을 **기술적 언어로 설명하는 연습 문서**  
- 기술 면접 대비보다는,  
  “이 개념을 얼마나 깊게 이해하고 논리적으로 설명할 수 있는가”에 초점  
- 구조 기반 설명력 향상을 위한 질의응답 형태의 정리

→ 학습한 개념을 명확하고 논리적으로 표현하기 위한 기술적 설명 연습 문서

# 🌱 학습 방식

1. Day 단위로 핵심 개념을 정리  
2. 개념을 실제 기술 언어로 설명하는 연습  
3. 유관 개념 간 비교와 연결 고리 파악  
4. 실무 환경에서 관련 이슈가 발생했을 때  
   어떤 원리로 접근해야 하는지에 대한 사고 방식 정립  

---

# 📌 이 저장소의 지향점

이 프로젝트는 **깊이 있는 이해와 실무 중심 사고를 기르는 데 초점**을 맞춥니다.

- 단순 요약이 아닌 “왜 그렇게 되는가(Why)”를 중심으로 정리  
- 컴퓨터 내부 동작 구조까지 내려가 설명  
- 현업 백엔드 개발자가 문제를 분석하고 해결하는 데 필요한 기술 기반 제공  
- 장기적으로도 변하지 않는 지식들 위주로 정리  

> “CS는 기술의 근본이며, 이 저장소는 근본을 단단히 하기 위한 개인 아카이브입니다.”
